BLOQUE 01conceptos
// el cerebro

Un LLM predice
la siguiente palabra.

Un Large Language Model es una red neuronal entrenada con cantidades enormes de texto. Aprende patrones estadísticos: dado un contexto, ¿qué token es más probable que venga después?

  • No "razona" como una persona — predice texto plausible.
  • No tiene memoria entre conversaciones (salvo lo que tú le des).
  • No sabe nada posterior a su fecha de entrenamiento.
  • Puede alucinar: decir falsedades con seguridad.
cómo funciona, simplificado
tokens
prompt:
"El Servicio de Informática mantiene"
predicciones del modelo:
→ "el" 42%
→ "Moodle" 28%
→ "los" 15%
→ "una" 9%
→ ...
elige uno · repite · genera texto
temperature, top-p, top-k controlan la elección
bloque 01 · 2/10
BLOQUE 01conceptos · tokens
// la unidad mínima

El modelo no ve palabras,
ve tokens.

Antes de entrar al modelo, tu texto se trocea en tokens: piezas de ~3-4 caracteres en español. El modelo predice tokens, no letras ni palabras enteras.

  • Una palabra corta suele ser 1 token; una larga, 2-3.
  • Acentos, emojis y código fuente gastan más tokens.
  • El contexto del modelo se mide en tokens (ej. 200k).
  • Pagas por tokens de entrada y de salida.
tokenizer · ejemplo
~ ≈ tokens
texto:
"Servicio de Informática de la UEx"
tokens:
[Servicio] [ de] [ Inform] [ática] [ de] [ la] [ U] [Ex]
→ 8 tokens · 33 caracteres
contexto típico:
Claude Sonnet 4.6 · 200k tokens ≈ 150k palabras
≈ un libro de ~500 páginas en una sola petición
menos tokens = más rápido y más barato
bloque 01 · 2b/10
BLOQUE 01conceptos
// del LLM al agente

Un agente es un LLM
con herramientas y un bucle.

El LLM por sí solo solo genera texto. Un agente le da capacidad de actuar: leer ficheros, ejecutar comandos, llamar APIs. Y un bucle que repite hasta terminar la tarea.

  • Modelo · el cerebro que decide.
  • Herramientas · lo que puede hacer (Read, Write, Bash…).
  • Bucle · razonar → actuar → observar → repetir.
  • Permisos · qué le dejas hacer sin preguntar.
el bucle del agente
razonar · actuar · observar
› objetivo: arregla el test que falla
⏵ piensa: necesito ver el test
⏵ Read(test/auth.test.js)
⏵ piensa: el error está en parseUser
⏵ Read(src/auth.js)
⏵ Edit(src/auth.js)
⏵ Bash(npm test)
✓ tests verdes · tarea completada

BLOQUE 02importante
// reglas no escritas

Qué NO debe entrar en un prompt.

  • Datos personales de alumnado, PDI, PAS (DNIs, expedientes, notas).
  • Configuraciones con credenciales, IPs internas, tokens.
  • Tickets con datos identificables.
  • ENS medio/alto y material clasificado.
  • Contratos, licitaciones, documentación legal.
  • Backups o dumps de bases de datos de producción.
Regla simple: si no lo mandarías por correo a un proveedor externo, no lo pegues en una IA.

bloque 04 · 1/8
BLOQUE 04idioma del prompt
// detalle que casi nadie cuenta

Habla con el LLM en inglés.

Los modelos están entrenados con muchísimo más texto en inglés que en español. El inglés es el idioma nativo del modelo — el español es una traducción aprendida.

razonamiento

Mejor en inglés

Más datos de entrenamiento → instrucciones más claras, menos ambigüedad y menos alucinaciones en tareas complejas.

tokens

Más barato y rápido

El mismo texto en español usa ~25-40% más tokens (acentos, palabras largas). En inglés pagas y esperas menos.

técnico

El código ya es inglés

APIs, librerías, errores, documentación, Stack Overflow… todo está en inglés. Hablar inglés con el modelo evita una traducción mental innecesaria.

Subagentes.

Claude puede invocarse a sí mismo igual que tú lo invocas. Útil para paralelizar, aislar contexto, o delegar.
